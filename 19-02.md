## Technical Details

### API Endpoints Used
1. `/auth/superadmin/create-super-admin` - Create super admin users
2. `/auth/admin/create-user` - Create regular users (admin only)
3. `/auth/users` - Fetch all users
4. `/tasks/assignedTo/${userId}` - Fetch tasks assigned to user
5. `/tasks/assignedBy/${userId}` - Fetch tasks assigned by user
6. `/tasks` - Create new tasks

### Authentication
- JWT tokens stored in `sessionStorage`
- Token passed in headers: `Authorization: Bearer ${token}`
- Role-based access control implemented

### Role Hierarchy
1. **Super Admin** - Can create other super admins
2. **Admin** - Can create users, manage user tasks
3. **User** - Regular task management
